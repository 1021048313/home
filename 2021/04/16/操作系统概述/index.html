操作系统-概述

  1. 可执行文件的文件格式常见有：

    exe 和 dll（Windows 系列）、elf（Linux 系列）和 Mach-O（Mac系列）；

  2. 操作系统定义

    计算机系统中的一个系统软件

    • CPU利用率
    • 资源管理
    • 用户界面和编程接口

  3. 操作系统功能

    • 自底向上：资源管理
      • 记录哪些空闲
      • 资源分配策略（算法）
        • 静态分配（执行前要多少给多少）-浪费
        • 动态分配（执行时要才给）
      • 资源分配和回收
      • 协调冲突、保护资源、提高利用率
      • 5个基本功能
        1. 进程/线程管理
        2. 存储管理
        3. 文件管理
        4. 设备管理
        5. 用户接口
    • 给用户提供服务
    • 扩展硬件机器

  4. 操作系统特征

    • 并发concurrency

      处理多个同时性活动

      程序在CPU上轮流执行，宏观表现为同时执行

      并行parallel：不同程序同时在多个硬件上执行

    • 共享sharing

      OS与多个用户的程序共同使用系统资源

      • 互斥共享（打印机）
      • 同时共享（文件读写）

    • 虚拟virtual

      物理实体映射为若干个对应的逻辑实体

      • 分时
      • 分空间

    • 随机

      对不可预测的次序发生的事件响应

  5. 典型操作系统架构

    • windows

      用户态、内核态

    • unix

      最里面是Unix内核，在它之上是Unix的系统调用，各种库文件是建立再系统调用之上的，可以由用户程序（application）调用，Shell是一个比较特殊的应用程序（application），它提供了运行其他用户程序的接口。用户程序（application）可以在Shell上运行，可以调用Library运行，也可以直接调用system calls 运行。

    • linux

      内核、shell、文件系统和应用程序

    • android

  6. OS分类

    • 批处理
    • 分时
    • 实时（硬实时、软实时）
    • 个人计算机操作系统
    • 网络操作系统
    • 分布式操作系统
    • 嵌入式操作系统

    TANENBAUM分类

  7. Spooling技术（假脱机技术）

    将一台独享打印机改造为可供多个用户共享的打印机，是应用 SPOOLing技术的典型实例。

    具体做法是：系统对于用户的打印输出，并不真正把打印机分配给该用户进程，而是先在输出井中申请一个空闲盘块区，并将要打印的数据送入其中；然后为用户申请并填写请求打印表，将该表挂到请求打印队列上。若打印机空闲，输出程序从请求打印队首取表，将要打印的数据从输出井传送到内存缓冲区，再进行打印，直到打印队列为空。
